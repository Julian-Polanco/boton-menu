Calidad en desarrollo de software

Definición de ISO

Organización internacional que regula una serie de normas para la fabricación,comercio y comunicación en la industria, generando asi la estandarización del producto.

Norma ISO/IEC 25000

¿Qué es la norma ISO/IEC 25000?

Es un conjunto de normas que tiene por objetivo la creación de un marco de trabajo común para evaluar la calidad del producto software.

¿Para qué sirve esta norma?

Esta norma sirve unificar y organizar dos procesos los cuales son la especificación de requisitos de calidad del software y evaluar la calidad de un proyecto de software.

Subdivisiones o características de esta norma

ISO Acción Función Características resaltantes
2500n División gestión de calidad Define todos los estándares o particularidades de la familia de las normas ISO/IEC 25000. Contiene todas las fases necesarias para llevar a cabo una evaluación de software.​
2501n División de Modelo de Calidad Contiene todas las normas y características de la calidad externa, interna y en uso del software. Representa las características y subcaracterísticas de calidad frente a las cuales se puede evaluar un proyecto de software. ​
2502n División de Medición de Calidad Contiene definiciones de las métricas internas, externas y de uso, también cuenta con estándares y una guía para su aplicación. Cuenta con un modelo de referencia de calidad del producto software.​
2503n División de Requisitos de Calidad Esta división nos ayuda a especificar los requisitos en cuestión de calidad. Pueden ser usados en el proceso de especificación en cuanto a la calidad de un producto software que va a ser desarrollado o como entrada para un proceso de evaluación. ​
2504n División de Evaluación de Calidad Nos brinda requisitos, recomendaciones y guías para poder llevar a cabo la evaluación del proyecto de software.​ incluye normas que proporcionan requisitos, recomendaciones y guías para llevar a cabo el proceso de evaluación del producto software.

Explicación audiovisual de la norma ISO/IEC 25000

ISO/IEC 9126

ISO 9126 era un estándar internacional que evaluaba la calidad del software. Este estándar fue reemplazado en 2005 por el conjunto de normas SQUARE ISO 25000 que también desarrolla los mismos conceptos.

¿Qué establece la norma ISO/IEC 9126?

La norma ISO/IEC 9126 permite evaluar y especificar la calidad del software desde diferentes criterios que son asociados con la adquisición, requerimientos, desarrollo, uso, evaluación, soporte, mantenimiento, aseguramiento de la calidad y auditoria del software.

Modelos de calidad establecidos por el estándar ISO 9126

Funcionalidad:

Permite calificar si un producto de software maneja de forma adecuada el conjunto de funciones que satisfagan las necesidades para las cuales fue diseñado.

Atributos:

adecuación, exactitud, interoperabilidad, conformidad, seguridad.

Confiabilidad:

Es la capacidad que tiene el software para mantener su nivel de ejecución bajo condiciones normales en un periodo de tiempo establecido.

Sub-características:

nivel de madurez, tolerancia a fallas, recuperación.

Usabilidad:

Permite evaluar el esfuerzo necesario que deberá invertir el usuario para utilizar el sistema.

Atributos:

compresibilidad, facilidad de aprender, operabilidad.

Eficiencia:

Permite evaluar la relación entre el nivel de funcionamiento del software y la cantidad de recursos usados.

Aspectos a evaluar:

Comportamiento con respecto al tiempo, comportamiento respecto a recursos.

Mantenibilidad:

Permite medir el esfuerzo necesario para realizar modificaciones al software, ya sea por la corrección de errores o por el incremento de funcionalidad.

Factores:

capacidad de análisis, capacidad de modificación, estabilidad, facilidad de prueba.

Portabilidad:

Es la habilidad del software para ser transferido de un ambiente a otro.

Aspectos:

adaptabilidad, facilidad de instalación, conformidad, capacidad de reemplazo.
